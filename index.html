What was most challenging about the project?

The most challenging part of project 1 was the trial and error component. I haven't had a lot of experience with responsive design coding so I was constantly editing the code with percentages and minimizing and maximizing the screen in order to see how to website worked on a laptop, then on a mobile device. It wasn't extremely difficult to code but it took a lot of time with trial and error in order to perfect the layout. Making the navbar responsive was probably the hardest part for me. I made the navbar absolute in the css in order for the body to not run over the top of the navbar. I'm not sure if it is responsive or not, but when I minimize the screen it stays in the same spot. I also had to google around in order to find out how to make the background responsive. It took me a few times before I figured it out.

Did you use any resources?

Yes I used quite a few resources. In order to get the grid based layout, I used responsivegridsystem.com. I entered the amount of rows and columns I desired and the website generated the code. Even though I copy and pasted the code, I still had to heavily edit the html and css to make it look like I wanted. This website was a good resource to save time and helped me understand grid based layouts better. I also incorporated google fonts and used the bootstrap website for coding purposes. For most of my help, I just googled what I needed and used random websites with trial and error in order to get my desired result. In order to make page 1 with the images, I used your example that you provided on your website. It took me longer than I thought to tweak the code and figure out how to make it work for my photographs. I struggled with all the divs at first and forgot to close some to it was quite a mess! However, I was able to understand columns and rows and how to lay photographs on a page.
